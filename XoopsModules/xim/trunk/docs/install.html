Installing Xoops Instant Messenger (xim)

First time installation

Preface:

Xim is a messenger system. Based on the jquery facebook chat example written by anant garg. The system has later been ported, adapted, and expanded to Xoops by Culex & Andrax.
The chat is one-on-one and fully supported by Xoops since version 2.4.0.

XIM is released under the terms of the GNU General Public License (GPL) and is free to use and modify. It is free to redistribute as long as you abide by the distribution terms of the GPL.

Requirements

  • WWW Server (Apache, IIS, Roxen, etc)
  • XOOPS 2.4.0 or higher as xim uses preloads
  • PHP 4.3.0 or higher (5.2 or higher recommended)
  • MySQL 3.23 or higher (4.1 or higher recommended)

recommended for testing xim

  • Browser Firefox and the Firefox plugin FireBug plugin (latest version)

Before you install

  1. Check to see if the downloaded xim version is the latest released version.
  2. Turn cookie and JavaScript of your browser on.

Installing on a hosted platform

  • Upload the unzipped folder xim to your module direcotry.
  • Install using XOOPS module install as any other module installation.
  • Make the block visible for the user groups you want to see the block (admin/moderators etc etc).

Common problems with XIM

I get blank page, can't click username, The block does not show.......

  • The block does only show if 2 or more users are logged in.
  • Make sure your block is visible on "All pages" and not only frontpage.
  • Make sure your Xoops version is > 2.4.0. Xoops versions before this does not use preloads.
  • Turn on debug and check to see if any errors emerge.
  • Using FireFox, right click on the screen, choose inspect element, choose console in the tabs and check to see if chat heartbeat is running
    "GET http://www.yoursite.com/modules/xim/chat.php?action=chatheartbeat&_=XXXXXXXXXXXX xxx OK xxx MS"
  • If no errors, use Firebug (Right click->inspect element->console) and see if any Jquery errors.
  • If no jquery errors. Clean caches-> module update xim, system modules.
  • If still blank page. Go to your protector module -> settings and write xim in the form "Modules out of Dos / Crawler checker".
  • Check if your theme is including more than one version of Jquery. Usually in the head of your theme.html file.
    Normally this is no problem as long as the other including jquery > 1.3.2
  • If still questions or errors please post at the Xoops support Forum
